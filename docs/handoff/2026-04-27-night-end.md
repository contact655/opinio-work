## 今夜実装した API の仕様サマリー

### PATCH /api/biz/members/[id]

| action | body | 動作 | ガード |
|--------|------|------|--------|
| `permission` | `{ value: "admin" \| "member" }` | permission 変更 | 自己降格防止 (403)、最後の admin 降格防止 (400) |
| `deactivate` | — | `is_active=false` | 自己無効化防止 (403)、最後の admin 無効化防止 (400) |
| `reactivate` | — | `is_active=true` | ガードなし |
| `update_profile` | `{ role_title?: string, department?: string }` | 役職・部署更新 | admin のみ (403)、空文字→null、100 文字上限 |

### POST /api/biz/members

```
{ email: string, permission: "admin" | "member" }
```

- 対象 email が ow_users に存在しない → 404
- すでにアクティブメンバー → 409
- 無効化済みレコードが存在 → UPDATE is_active=true + permission（201 で成功扱い）
- 新規 → INSERT（201）
